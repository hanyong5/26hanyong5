# hanyong5 회사홈페이지 — 요구사항 명세서

## 프로젝트 목표

기업형 홈페이지 구축 (React 19 + Supabase + Tailwind CSS)

---

## 요구사항 목록

### REQ-01. 회원가입

- 이메일 + 비밀번호 입력으로 가입
- React Hook Form + Zod 유효성 검사 적용
- Supabase Auth 연동
- 가입 완료 시 `profiles` 테이블에 사용자 레코드 자동 생성
- 이메일 중복 가입 방지

---

### REQ-02. 로그인 / 로그아웃

- 이메일 + 비밀번호 로그인
- 로그인 성공 시 세션 유지 (Supabase Auth 세션)
- 로그아웃 시 세션 완전 파기
- 로그인 상태에 따라 Header 메뉴 동적 변경 (로그인/로그아웃 버튼)

---

### REQ-03. 권한 분리 (Protected Route / Admin Route)

- `user` / `admin` 두 가지 역할 구분
- 비로그인 사용자가 보호된 페이지 접근 시 로그인 페이지로 리다이렉트
- `admin` 역할이 아닌 사용자가 관리자 페이지 접근 시 접근 거부
- `profiles.role` 컬럼으로 역할 관리

---

### REQ-04. 회사소개 다중 페이지

- 아래 5개 서브페이지 구성
  - 회사소개
  - CEO 인사말
  - 비전
  - 연혁 (타임라인 UI)
  - 오시는 길 (지도 또는 주소 표시)
- 좌측 또는 상단 서브 내비게이션 제공

---

### REQ-05. 제품 목록

- 카드형 UI로 제품 표시 (이미지 + 이름 + 요약)
- 카테고리 필터 (전체 / 카테고리별)
- 제품명 키워드 검색
- 페이지네이션 적용

---

### REQ-06. 제품 상세

- 제품 이미지, 상세 설명 표시
- "문의하기" 버튼 — 클릭 시 온라인문의 페이지로 이동
- Supabase Storage `products/` 버킷에서 이미지 로드

---

### REQ-07. 커뮤니티 게시글 목록

- 제목 + 작성자 + 작성일 + 조회수 표시
- 제목 키워드 검색
- 페이지네이션 적용
- 공지 게시글 상단 고정 표시

---

### REQ-08. 커뮤니티 게시글 상세

- 본문 내용 표시
- 페이지 진입 시 조회수 1 증가
- 첨부파일 다운로드 링크 표시

---

### REQ-09. 커뮤니티 게시글 CRUD

- 작성: 로그인 사용자만 가능, 에디터 + 파일 첨부 지원
- 수정: 본인 게시글만 가능
- 삭제: 본인 게시글만 가능 (관리자는 전체 삭제 가능)
- 첨부파일은 Supabase Storage `editor/` 버킷에 저장

---

### REQ-10. 온라인문의 작성

- 로그인 사용자만 문의 작성 가능
- React Hook Form + Zod 유효성 검사 적용
- 문의 유형 선택 (제품문의 / 일반문의 / 기타)
- 제목 + 내용 필수 입력

---

### REQ-11. 온라인문의 내역 조회

- 본인이 작성한 문의만 조회 (RLS 정책으로 강제)
- 문의 상태 표시: 접수중 / 답변완료
- 관리자 답변 내용 표시

---

### REQ-12. 관리자 — 회원 관리

- 전체 회원 목록 조회
- 회원 역할 변경 (user ↔ admin)
- 회원 계정 비활성화 처리

---

### REQ-13. 관리자 — 콘텐츠 관리

- **제품 관리**: 제품 등록 / 수정 / 삭제, 이미지 업로드
- **게시판 관리**: 전체 게시글 관리, 공지 설정 / 해제
- **문의 관리**: 문의 목록 조회, 답변 작성, 상태 변경 (접수중 → 답변완료)
- **사이트 설정**: 회사 기본 정보 관리

---

### REQ-14. 공통 레이아웃

- Header: 로고, 주요 메뉴, 로그인/로그아웃 버튼
- Footer: 회사정보, 저작권
- Navigation: 메인 메뉴 (회사소개 / 제품소개 / 커뮤니티 / 온라인문의)
- Breadcrumb: 현재 위치 표시
- 반응형 레이아웃 (모바일 / 태블릿 / 데스크톱)

---

### REQ-15. 보안 — RLS 및 Storage 정책

- 모든 신규 테이블에 Row Level Security(RLS) 활성화
- 테이블별 RLS 정책 정의
  - `profiles`: 본인 읽기/수정, admin 전체 접근
  - `products`: 전체 읽기, admin만 쓰기
  - `posts`: 전체 읽기, 로그인 사용자 작성, 본인/admin 수정·삭제
  - `inquiries`: 본인 읽기/작성, admin 전체 접근
- Supabase Storage 버킷별 접근 정책 설정
  - `products/`: public 읽기, admin만 쓰기
  - `company/`: public 읽기, admin만 쓰기
  - `editor/`: 로그인 사용자 읽기/쓰기
