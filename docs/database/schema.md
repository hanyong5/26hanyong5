# 데이터베이스 스키마

> 실제 테이블 생성 전 MCP로 DB를 확인한 후 이 문서를 기준으로 작업한다.

---

## ERD 관계도

```
auth.users (Supabase 제공)
    │
    ▼
profiles          ── posts
(id = auth.uid)   ── inquiries

products          (독립 테이블)
```

---

## 테이블 정의

### profiles

회원 프로필 — `auth.users` 와 1:1 연동

| 컬럼 | 타입 | 제약 | 설명 |
|------|------|------|------|
| id | uuid | PK, FK → auth.users.id | Supabase Auth UID |
| email | text | NOT NULL | 이메일 |
| name | text | | 이름 |
| phone | text | | 연락처 |
| role | text | DEFAULT 'user' | 역할 (user / admin) |
| avatar_url | text | | 프로필 이미지 URL |
| created_at | timestamptz | DEFAULT now() | 가입일 |
| updated_at | timestamptz | DEFAULT now() | 수정일 |

**RLS 정책**
- 본인 레코드 읽기/수정 허용
- admin은 전체 읽기/수정 허용

**트리거**
- `auth.users` INSERT 시 `profiles` 자동 생성

```sql
CREATE TABLE profiles (
  id         uuid PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  email      text NOT NULL,
  name       text,
  phone      text,
  role       text NOT NULL DEFAULT 'user' CHECK (role IN ('user', 'admin')),
  avatar_url text,
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);

ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;

-- 본인 읽기
CREATE POLICY "profiles_select_own" ON profiles
  FOR SELECT USING (auth.uid() = id);

-- 본인 수정
CREATE POLICY "profiles_update_own" ON profiles
  FOR UPDATE USING (auth.uid() = id);

-- admin 전체 읽기
CREATE POLICY "profiles_select_admin" ON profiles
  FOR SELECT USING (
    EXISTS (SELECT 1 FROM profiles WHERE id = auth.uid() AND role = 'admin')
  );
```

### products

제품소개 — 관리자가 등록/수정/삭제

| 컬럼 | 타입 | 제약 | 설명 |
|------|------|------|------|
| id | bigint | PK, IDENTITY | 제품 ID |
| name | text | NOT NULL | 제품명 |
| category | text | | 카테고리 |
| summary | text | | 요약 설명 |
| content | text | | 상세 설명 |
| image_url | text | | 대표 이미지 URL (Storage: products/) |
| is_featured | boolean | DEFAULT false | 메인 노출 여부 |
| sort_order | integer | DEFAULT 0 | 정렬 순서 |
| created_at | timestamptz | DEFAULT now() | 등록일 |
| updated_at | timestamptz | DEFAULT now() | 수정일 |

**RLS 정책**
- 전체 읽기 허용 (비로그인 포함)
- admin만 INSERT / UPDATE / DELETE 허용

```sql
CREATE TABLE products (
  id          bigint PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  name        text NOT NULL,
  category    text,
  summary     text,
  content     text,
  image_url   text,
  is_featured boolean DEFAULT false,
  sort_order  integer DEFAULT 0,
  created_at  timestamptz DEFAULT now(),
  updated_at  timestamptz DEFAULT now()
);

ALTER TABLE products ENABLE ROW LEVEL SECURITY;

-- 전체 읽기
CREATE POLICY "products_select_all" ON products
  FOR SELECT USING (true);

-- admin 쓰기
CREATE POLICY "products_write_admin" ON products
  FOR ALL USING (
    EXISTS (SELECT 1 FROM profiles WHERE id = auth.uid() AND role = 'admin')
  );
```

### posts

커뮤니티 게시판

| 컬럼 | 타입 | 제약 | 설명 |
|------|------|------|------|
| id | bigint | PK, IDENTITY | 게시글 ID |
| user_id | uuid | FK → profiles.id | 작성자 |
| title | text | NOT NULL | 제목 |
| content | text | NOT NULL | 본문 |
| is_notice | boolean | DEFAULT false | 공지 여부 |
| view_count | integer | DEFAULT 0 | 조회수 |
| file_url | text | | 첨부파일 URL (Storage: editor/) |
| file_name | text | | 첨부파일 원본명 |
| created_at | timestamptz | DEFAULT now() | 작성일 |
| updated_at | timestamptz | DEFAULT now() | 수정일 |

**RLS 정책**
- 전체 읽기 허용 (비로그인 포함)
- 로그인 사용자 INSERT 허용
- 본인 또는 admin만 UPDATE / DELETE 허용

```sql
CREATE TABLE posts (
  id          bigint PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  user_id     uuid REFERENCES profiles(id) ON DELETE SET NULL,
  title       text NOT NULL,
  content     text NOT NULL,
  is_notice   boolean DEFAULT false,
  view_count  integer DEFAULT 0,
  file_url    text,
  file_name   text,
  created_at  timestamptz DEFAULT now(),
  updated_at  timestamptz DEFAULT now()
);

ALTER TABLE posts ENABLE ROW LEVEL SECURITY;

-- 전체 읽기
CREATE POLICY "posts_select_all" ON posts
  FOR SELECT USING (true);

-- 로그인 사용자 작성
CREATE POLICY "posts_insert_auth" ON posts
  FOR INSERT WITH CHECK (auth.uid() IS NOT NULL);

-- 본인 수정/삭제
CREATE POLICY "posts_update_own" ON posts
  FOR UPDATE USING (auth.uid() = user_id);

CREATE POLICY "posts_delete_own" ON posts
  FOR DELETE USING (auth.uid() = user_id);

-- admin 전체 수정/삭제
CREATE POLICY "posts_all_admin" ON posts
  FOR ALL USING (
    EXISTS (SELECT 1 FROM profiles WHERE id = auth.uid() AND role = 'admin')
  );
```

### inquiries

온라인문의

| 컬럼 | 타입 | 제약 | 설명 |
|------|------|------|------|
| id | bigint | PK, IDENTITY | 문의 ID |
| user_id | uuid | FK → profiles.id | 문의자 |
| type | text | DEFAULT 'general' | 문의 유형 (product / general / other) |
| title | text | NOT NULL | 제목 |
| content | text | NOT NULL | 문의 내용 |
| status | text | DEFAULT 'pending' | 상태 (pending / answered) |
| answer | text | | 관리자 답변 |
| answered_at | timestamptz | | 답변 일시 |
| created_at | timestamptz | DEFAULT now() | 문의일 |
| updated_at | timestamptz | DEFAULT now() | 수정일 |

**RLS 정책**
- 본인 문의만 읽기 허용
- 로그인 사용자 INSERT 허용
- admin은 전체 읽기 / 답변(UPDATE) 허용

```sql
CREATE TABLE inquiries (
  id          bigint PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  user_id     uuid REFERENCES profiles(id) ON DELETE SET NULL,
  type        text DEFAULT 'general' CHECK (type IN ('product', 'general', 'other')),
  title       text NOT NULL,
  content     text NOT NULL,
  status      text DEFAULT 'pending' CHECK (status IN ('pending', 'answered')),
  answer      text,
  answered_at timestamptz,
  created_at  timestamptz DEFAULT now(),
  updated_at  timestamptz DEFAULT now()
);

ALTER TABLE inquiries ENABLE ROW LEVEL SECURITY;

-- 본인 읽기
CREATE POLICY "inquiries_select_own" ON inquiries
  FOR SELECT USING (auth.uid() = user_id);

-- 로그인 사용자 작성
CREATE POLICY "inquiries_insert_auth" ON inquiries
  FOR INSERT WITH CHECK (auth.uid() IS NOT NULL);

-- admin 전체 접근
CREATE POLICY "inquiries_all_admin" ON inquiries
  FOR ALL USING (
    EXISTS (SELECT 1 FROM profiles WHERE id = auth.uid() AND role = 'admin')
  );
```

| 버킷 | 경로 | 용도 | 접근 |
|------|------|------|------|
| products | `products/{filename}` | 제품 이미지 | public 읽기 / admin 쓰기 |

## Auth 트리거 (자동 profiles 생성)

```sql
CREATE OR REPLACE FUNCTION public.handle_new_user()
RETURNS trigger AS $$
BEGIN
  INSERT INTO public.profiles (id, email, name)
  VALUES (
    NEW.id,
    NEW.email,
    NEW.raw_user_meta_data->>'name'
  );
  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW EXECUTE FUNCTION public.handle_new_user();
```
